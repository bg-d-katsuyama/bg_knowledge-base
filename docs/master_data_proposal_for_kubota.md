## 1. 背景

ご要望「土壌 R&D に絞ったマスターデータを Google スプレッドシートで作成したい」を踏まえ、Notion ナレッジベース（KB DB、3,024 件）を分析し、設計のたたき台を作成しました。

KB を分析した結果、**土壌関連タグ付きのエントリは 1,099 件（KB 全体の 36%）**あり、マスターデータのソースとして十分なボリュームが確認できました。

---

## 2. 協議材料（3 点）

### ① 設計たたき台
`docs/master_data_schema.md`

「1 行が何を表すか」について 3 案を比較しています。

| 案 | 粒度 | 想定行数 | コスト | 推奨度 |
|---|---|---|---|---|
| A | 1 行 = 1 KB エントリ | 1,099 行 | $0 | △ |
| **B** | **1 行 = 1 知見** | **3,000〜8,000 行** | **$10〜30** | **◎（推奨）** |
| C | 2 シート分割（知見+出典） | 同上 | $10〜30 | ○ |

### ② 実データサンプル
`docs/master_data_sample.md`

案 B で実際に 5 KB エントリから 17 知見を抽出した結果です。

**列構成**: 知見タイトル / 内容 / 領域分類 / 関連タグ / 発言者 / 所属 / 信頼度 / 検証ステータス / 発生日 / 出典 URL

**抽出例**:
- 「バイムフード堆肥化における放線菌の役割と指標性」（微生物・中・根拠あり）
- 「いかす資料における可給態窒素の記載欠如」（モニタリング・高・根拠あり）
- 「東京農業大学は Soil Quality Index 日本版に関心」（モニタリング・中・仮説）

### ③ CSV（Excel/Sheets で開ける形式）

## 3. ご判断いただきたい点（7 項目）

| # | 確認事項 | 選択肢 |
|---|---|---|
| 1 | **粒度の選定** | 案 A / 案 B（推奨）/ 案 C |
| 2 | **領域分類**の確定 | 微生物 / ミネラル / 堆肥 / LCA / pH / モニタリング / その他 の 7 区分で過不足ないか |
| 3 | **業務進捗系の取り扱い** | 例「資料更新が未完了」のような業務報告もマスターデータに含めるか、フィルタで除外するか |
| 4 | **対象タグの絞り込み** | 土壌関連 253 タグ（CSV あり）すべてを対象にするか、絞り込みたいタグはあるか |
| 5 | **追加取り込み対象の確認** | 共有された Notion ページ「腐植性堆肥製造マニュアル_v1.3」は KB 未取り込みのため取り込み対象に追加する想定でよいか |
| 6 | **書き込み先 Sheets** | 新規スプレッドシートを作成するか、既存スプレッドシートを指定するか |
| 7 | **公開範囲** | 久保田様のみ閲覧 / 特定メンバーに共有 / チーム全員 |
